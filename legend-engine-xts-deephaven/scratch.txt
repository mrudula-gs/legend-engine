The "data" directory is relative to wherever your docker-compose.yml file is (which should also be the directory from which you run 'docker-compose up'). If you are using straight docker(/podman), and not running with docker-compose, you can also just set the mount path explicitly, such as:
docker run --rm --name deephaven -p 10000:10000 --mount "type=bind,source=/tmp/dh-data,target=/data" --env START_OPTS="-Dauthentication.psk=my-secret-key -Ddeephaven.console.type=groovy" ghcr.io/deephaven/server:latest
Depending on the system configuration, on RHEL you may need to change the SELinux context of the directory to be mounted, which can be accomplished with --volume (instead of --mount) and the "Z" mount option (which changes SELinux labels for you):
mkdir /tmp/dh-data
docker run --rm --name deephaven -p 10000:10000 --volume "/tmp/dh-data:/data:Z" --env START_OPTS="-Dauthentication.psk=my-secret-key -Ddeephaven.console.type=groovy" ghcr.io/deephaven/server:latest
